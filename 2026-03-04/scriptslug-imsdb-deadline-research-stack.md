# Script Slug + IMSDb + Deadline：编剧与导演的“信息铁三角”

> **TL;DR**: 如果 ShotDeck/Frameset/Flim 解决的是“画面参考”，那 Script Slug、IMSDb、Deadline 解决的是另一个更底层问题：**故事文本 + 行业动态**。Script Slug 和 IMSDb 提供剧本学习素材，Deadline 提供项目与产业风向。三者合在一起，就是“写什么 + 怎么拍 + 市场现在要什么”的闭环。

---

## 为什么这三个站应该一起看？

大多数创作者的问题不是不会拍，而是：

```
1) 剧本结构怎么学？
2) 行业现在在买什么类型？
3) 我的题材有没有市场窗口？
```

这三个网站刚好对应这三件事。

---

## 1) Script Slug：更现代的剧本学习入口

**定位**：面向教育与创作参考的 screenplay 平台（强调 fair use 教学用途）

### 优势
- 界面和检索体验通常比老站更现代
- 偏“学习与拆解”场景
- 有 RSS（脚本/文章）方便持续追踪

### 适合人群
- 正在练习写作结构的新编剧
- 需要快速找某类型剧本参考的短视频团队
- 做 AI 剧本微调的人（先人工拆结构）

---

## 2) IMSDb：老牌剧本数据库（量大）

**定位**：互联网经典电影剧本库，按字母和类型检索，覆盖广

### 优势
- 库大，历史沉淀深
- 可直接在线阅读 HTML 版
- 类型分类清楚（Action/Drama/Sci-Fi…）

### 局限
- 站点体验偏老旧
- 元数据结构和质量不总是统一

### 适合人群
- 需要大量横向阅读的写作者
- 研究“同类型片怎么写”的人
- 要做脚本语料手工筛选的人

---

## 3) Deadline：行业情报雷达

**定位**：好莱坞产业新闻与项目动态（融资、并购、发行、奖季、选角）

### 优势
- 更新快，行业信号强
- 对“什么题材正在被买/被推”很敏感
- 能看到 studio 层面的产能与策略变化

### 典型价值
```
你写了一个本子，不确定市场要不要：
  → 去 Deadline 看最近同类项目是否在升温
  → 看发行策略、片厂产量、平台偏好
  → 再决定你是做长片 pitch 还是短剧化切入
```

---

## 实战工作流（给 QCut/Director Panel 用户）

```
第一步：Script Slug / IMSDb
  → 找 3-5 个同类型剧本
  → 拆：开场钩子、转折点、人物弧线、对白密度

第二步：Deadline
  → 查同题材近期新闻（融资/发行/口味变化）
  → 判断市场窗口和受众偏好

第三步：QCut Director Panel
  → 把结构映射成分镜任务
  → 先做 proof-of-concept（30-60秒）
  → 数据反馈后扩成完整项目
```

---

## 它们和 ShotDeck/Frameset/Flim 的关系

不是替代关系，而是前后链路：

```
Script Slug / IMSDb / Deadline  → 文字与市场层
ShotDeck / Frameset / Flim      → 视觉与提案层
QCut                             → 制作执行层
```

你可以理解为：
- 前者决定“讲什么、卖给谁”
- 中者决定“看起来像什么”
- 后者决定“怎么高效做出来”

---

## 🦞 龙虾结论

如果你现在只选一个动作：

1. 去 Script Slug / IMSDb 读 10 个同题材剧本
2. 去 Deadline 看近 30 天同类项目动态
3. 再开 QCut 做第一个 60 秒样片

你会比“直接开干”少走很多弯路。
