# Chrome 146 Ships Native MCP Support: One Toggle to Give AI Agents Your Browser

> Source: [@xpasky](https://x.com/xpasky/status/2032252486145253865) | 551 likes, 36 RT | 2026-03-13

## What Happened

Chrome 146 shipped yesterday with built-in MCP (Model Context Protocol) support. The practical impact:

**With a single toggle in Chrome settings, you can expose your live browsing session via MCP to any CLI agent.**

Your agent operates within your existing login sessions. No separate auth flows, no cookie exports, no headless browser setups. If you're logged into Gmail in Chrome, your agent can work with your email directly.

## Why This Matters

Before this, getting an AI agent to operate a browser required:
1. Setting up headless Chrome + CDP
2. Managing auth state (cookie exports, OAuth configs)
3. Maintaining a separate browser automation environment

Now? **One toggle. Done.**

Petr Baudis called this back in January 2026 when Chrome 144 was in early stable:

> "I should be able to just `amp mcp add chrome-devtools -- npx chrome-devtools-mcp@latest --autoConnect` and let Claude browse on my behalf, within my login sessions."

Two months later, Chrome 146 made it real.

## How It Works

From the attached screenshots, Chrome's settings now include MCP options. Once enabled, it exposes an MCP endpoint that CLI agents (Claude Code, Codex, etc.) can connect to via the standard MCP protocol:

- Process LinkedIn connection requests within your logged-in session (exactly what the author did)
- Search and reply to emails in Gmail
- Execute actions in any SaaS tool you're already authenticated with

## Builder Takeaways

1. **Auth is solved** — The user's browser session IS the auth. No more per-site login flows for your agent.
2. **MCP is now browser infrastructure** — Official Chrome support means this isn't experimental. It's a platform feature.
3. **Agent capability boundary expands massively** — Anything you can do in Chrome, your agent can now do too.

## Caveats

- Requires Chrome 146+ — make sure you're updated
- Security implications are real: you're exposing your entire browser session. Only use with trusted agents
- Currently developer/power-user oriented; mainstream UX will follow

## Bottom Line

Native Chrome MCP = AI agents get "human-level" browser access. Not simulated, not headless — your actual logged-in sessions. This is an infrastructure milestone for the agent ecosystem.
