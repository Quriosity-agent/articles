# Context7 CLI: Any AI Agent Can Now Pull Latest Docs — No MCP Required

> Based on [@enesakar](https://x.com/enesakar)'s [tweet](https://x.com/enesakar/status/2031887459085377887) | 2026-03-14

## TL;DR

Context7 launched CLI mode — one command (`npx ctx7 setup`) lets any AI coding agent pull up-to-date, version-specific library documentation directly into prompts. No MCP server needed.

## The Problem

LLMs writing code hit three recurring issues:

1. **Stale docs** — training data can be a year old
2. **Hallucinated APIs** — functions that don't exist
3. **Version mismatch** — answers for the wrong package version

Context7 solves this by injecting current, version-matched documentation and code examples straight into your prompt context. Previously this required an MCP server. Now there's a **CLI + Skills** mode that works with any agent.

## How It Works

```bash
npx ctx7 setup
```

This single command:
- Authenticates via OAuth
- Generates an API key
- Installs the appropriate skill for your coding agent

You can choose between CLI + Skills or MCP mode, and target a specific agent with `--cursor`, `--claude`, or `--opencode`.

### Core Commands

- `ctx7 library <name>` — Search the Context7 index for libraries
- `ctx7 docs <library-id>` — Fetch docs for a specific library (e.g., `/mongodb/docs`, `/vercel/next.js`)
- `resolve-library-id` — Resolve a library name to a Context7-compatible ID

### In-Prompt Usage

```
Implement basic auth with Supabase. use library /supabase/supabase for API and docs.
```

The slash syntax tells Context7 exactly which library to load — skipping the search step entirely.

## Why CLI Mode Matters

| | MCP Mode | CLI + Skills Mode |
|---|---|---|
| Requires MCP Server | ✅ | ❌ |
| Works with any agent | Limited | ✅ |
| Setup complexity | Moderate | One command |
| Works without MCP support | ❌ | ✅ |

The key insight: not every AI agent supports MCP. CLI mode turns documentation retrieval into standard shell commands + skill triggers. **Any agent that can run shell commands can use Context7.**

The announcement pulled **1,004 likes, 96 retweets, and 40 replies** — exceptional engagement that signals strong demand for solving the "stale docs in AI coding" problem.

## Practical Recommendations

1. **Already using MCP?** Keep using it — CLI is an alternative, not a replacement
2. **Agent doesn't support MCP?** CLI mode is your answer — `npx ctx7 setup`
3. **Auto-trigger doc fetching:** Add a rule to CLAUDE.md or Cursor Rules: "Always use Context7 when I need library/API documentation"
4. **Version-specific docs:** Just mention the version in your prompt — Context7 handles the matching
