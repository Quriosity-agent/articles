## 背景

0xKenny 盘点了过去两周 GitHub 上增长最快的 AI 开源项目，其中 **Agency Agents** 拿到了 35K+ Star，成为最亮眼的项目之一。

## Agency Agents 是什么

一句话：**把 Claude Code 一键变成 51 位 AI 专家 + 9 大部门的完整团队。**

覆盖的部门包括：
- 🏗️ **产品** — 需求分析、PRD 编写、用户故事
- 💻 **开发** — 前后端、架构设计、代码审查
- 🎨 **设计** — UI/UX、设计系统、原型
- 📊 **营销** — 内容策略、SEO、增长
- 📱 **社媒** — 社交媒体运营、内容发布
- 🧪 **QA** — 测试策略、自动化测试、Bug 追踪
- 📈 **数据** — 分析、报告、洞察
- 🔒 **安全** — 安全审计、合规
- 📋 **项目管理** — 进度跟踪、资源协调

简单说就是：一个人 + Claude Code + Agency Agents = 一整家公司的执行力。

## 为什么值得关注

1. **降低门槛**：不需要复杂的 multi-agent 框架配置，基于 Claude Code 的 skill/persona 系统直接用
2. **全流程覆盖**：从产品到部署到运维，不是只做开发那一环
3. **35K+ Star 说明市场需求**：solo developer 和小团队确实需要这种"虚拟公司"能力

## Builder 视角

对于实际在做项目的人来说，这个项目的核心价值不是"替代团队"，而是：

- **快速原型**：一个人也能走完完整的产品流程
- **质量兜底**：QA、安全这些容易被 solo dev 忽略的环节有了覆盖
- **知识补盲**：你不懂营销？让营销部门的 agent 帮你想策略

但也要注意：51 个 expert 的 prompt 质量参差不齐是常见问题，实际使用中大概率需要根据自己的场景做调整。

## 小结

Agency Agents 代表了一个趋势：**AI 工具正在从"单点能力"向"组织级能力"演进。** 不再是"帮我写个函数"，而是"帮我运营一家公司"。

对于 Builder 来说，值得 clone 下来研究一下它的 prompt 设计和角色划分逻辑——这比项目本身更有学习价值。
