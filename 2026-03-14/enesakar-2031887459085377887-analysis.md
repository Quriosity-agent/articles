# Context7 CLI：让任何 AI Agent 都能拉取最新文档，不再依赖 MCP

> 基于 [@enesakar](https://x.com/enesakar) 的 [推文](https://x.com/enesakar/status/2031887459085377887) | 2026-03-14

## 一句话总结

Context7 发布了 CLI 模式——只需一行命令 `npx ctx7 setup`，任何 AI agent 都能直接拉取最新版本的库文档，不再需要 MCP 服务器。

## 为什么这很重要

LLM 写代码时有三个老问题：

1. **文档过时** — 训练数据可能是一年前的
2. **API 幻觉** — 编出根本不存在的函数
3. **版本混乱** — 给你老版本的用法

Context7 的方案是：在 prompt 里直接注入最新的、版本匹配的官方文档和代码示例。之前只有 MCP 模式（需要注册 MCP Server），现在新增了 **CLI + Skills** 模式。

## CLI 模式怎么用

```bash
npx ctx7 setup
```

这一行命令会：
- 通过 OAuth 认证
- 生成 API key
- 自动安装对应的 skill（给你的 coding agent 用）

你可以选择 CLI + Skills 模式或者 MCP 模式，还能用 `--cursor`、`--claude`、`--opencode` 指定目标 agent。

### 核心命令

- `ctx7 library <名称>` — 搜索 Context7 索引里的库
- `ctx7 docs <库ID>` — 拉取指定库的文档（比如 `/mongodb/docs`、`/vercel/next.js`）
- `resolve-library-id` — 把库名解析成 Context7 兼容的 ID

### 在 prompt 里用

```
用 Supabase 实现基础认证。use library /supabase/supabase for API and docs.
```

斜杠语法直接告诉 Context7 去拉哪个库的文档，跳过匹配步骤。

## 为什么 CLI 模式比 MCP 更灵活

| | MCP 模式 | CLI + Skills 模式 |
|---|---|---|
| 需要 MCP Server | ✅ | ❌ |
| 适用于任何 agent | 受限 | ✅ |
| 设置复杂度 | 中等 | 一行命令 |
| 适合无 MCP 支持的工具 | ❌ | ✅ |

关键点：不是所有 AI agent 都支持 MCP。CLI 模式把文档获取变成了普通的命令行操作 + skill 触发，**任何能执行 shell 命令的 agent 都能用**。

## 实战建议

1. **已经用 MCP 的**：不用换，MCP 继续用就行
2. **Agent 不支持 MCP 的**：CLI 模式是救星，`npx ctx7 setup` 搞定
3. **想自动触发文档拉取**：在 CLAUDE.md 或 Cursor Rules 里加一条规则，比如 "Always use Context7 when I need library/API documentation"
4. **指定版本**：直接在 prompt 里说版本号，Context7 会自动匹配
