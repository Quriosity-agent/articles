## 这东西解决什么问题？

用过 AI Agent 的人都知道这几个痛点：

1. **聊完就忘** — 每次对话都要重新喂上下文，Agent 没有长期记忆
2. **Token 烧钱** — 无效内容占了大半，每次都把全量上下文塞进去
3. **检索是黑盒** — 不知道 Agent 为什么给出离谱答案，没法调试
4. **技能不持久** — 教过的规则转头就忘，手动重复

OpenViking 是字节跳动火山引擎 Viking 团队开源的 **Context Database**，专门给 AI Agent 装一个"不会失忆的大脑"。

## 核心设计：文件系统范式

OpenViking 最聪明的地方：**不搞复杂的向量数据库，直接用你熟悉的文件系统来组织上下文。**

通过统一的 `viking://` 协议，所有内容像电脑文件一样归档：

```
viking://user/memories/     → 用户记忆
viking://agent/skills/      → Agent 技能
viking://resources/          → 外部资源
```

建文件夹、搜文件、定位内容 — 全是你熟悉的操作。不用学向量数据库，不用理解 embedding 原理。

## 三级存储：Token 账单直接砍 70%

这是最实在的部分。每段上下文自动生成三级存储，按需加载：

| 层级 | 内容 | Token 数 | 什么时候加载 |
|------|------|----------|-------------|
| **L0** | 一句话摘要 | ~100 | 快速浏览定位 |
| **L1** | 核心内容 | ~2000 | 需要细节时 |
| **L2** | 完整原文 | 完整 | 绝对必要时 |

Agent 先用 L0 快速扫一遍，找到相关的再加载 L1，只有真正需要原文时才加载 L2。不用每次把全量上下文全塞进去。

## 自进化能力

每次对话结束，OpenViking 自动从交互中提取长期记忆。不用重新训练模型，不用手动更新规则。Agent 越用越懂你。

## 可视化检索轨迹

Agent 检索了什么内容、为什么加载这个资源 — 全链路可视化。调试 Agent 不再是猜谜游戏。

## 上手

```bash
pip install openviking
```

支持多种 VLM 提供商（Volcengine、OpenAI、LiteLLM），Embedding 模型也是即插即用。

项目兼容 OpenClaw 等主流 Agent 框架，GitHub 发布不到 3 个月已经 5.4K+ Star。

## 技术背景

这个团队从 2019 年起就一直给 TikTok 和字节全业务提供向量搜索技术支持。底层技术在亿级流量里跑了 7 年，不是实验室产物。

## Builder 视角

**值得关注的点：**
- 文件系统范式降低了认知负担，不需要理解向量数据库就能管理 Agent 上下文
- L0/L1/L2 分层加载是真省钱的设计，对长会话场景特别有用
- 可视化检索轨迹对调试 Agent 行为非常关键

**需要注意的：**
- 需要 Python 3.10+、Go 1.22+、C++ 编译器，依赖链不算轻
- 自进化记忆的质量取决于 VLM 的理解能力，模型选型很重要
- 目前主要是中文社区驱动，英文文档还在完善中
