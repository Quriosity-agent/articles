# 110K Stars Consensus: Three SDD Tools Reshaping AI Coding

> **TL;DR**: The core AI coding problem isn't "can AI write code" but "how to make AI follow your intent." Three SDD (Spec-Driven Development) tools with 110K+ combined stars: **Spec-Kit** (GitHub official, 69.1K⭐) = "what rules to follow," **OpenSpec** (23.7K⭐) = "what changed," **Superpowers** (50K⭐) = "how to execute."

## Spec-Kit: GitHub's "Project Constitution"
5-step workflow: constitution → specify (what) → plan (how) → tasks → implement. Key: separates requirements from implementation to prevent AI scope drift.

## OpenSpec: Flexible Change Management
Action-based OPSX workflow: new → continue → apply → archive. Key: archive system gives AI "memory" of past decisions.

## Superpowers: Engineer-Level Execution
Not documentation — it's **methodology**. Key skills: TDD-first, sub-agent driven development (each task gets independent agent + 2-phase review), systematic debugging, git worktrees.

## How They Work Together
- **Spec-Kit**: Global governance (constitution)
- **OpenSpec**: Per-change lifecycle
- **Superpowers**: Execution quality (TDD + code review)

## Why 110K Stars?
Developers realized: **AI's coding ability isn't the bottleneck — making AI understand your intent is.** SDD tools are the answer.
