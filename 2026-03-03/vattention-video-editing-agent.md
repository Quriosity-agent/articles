# Vattention（时空注力）：视频编辑的 Claude Code 时刻来了？

> **TL;DR**: 视频后期 AI Agent 创业公司 **Vattention（时空注力）** 完成数百万美元种子轮融资（青锐资本、百度风投、常垒资本）。核心主张：视频编辑正从 **PostPro 2.0（非线性剪辑）** 进入 **PostPro 3.0（指数剪辑）**。类比编程领域从 Copilot → Claude Code 的范式转变，Vattention 要做视频后期的 **Video-Editing Agent**。三大引擎 MACE/ACE/PACE，核心是 **Context Modeling** — 理解创作者意图，不是简单的一键生成。

---

## 🎬 核心洞察：PostPro 1.0 → 2.0 → 3.0

```
PostPro 1.0: 线性剪辑     → 按顺序操作（磁带时代）
PostPro 2.0: 非线性剪辑   → 随机访问（Premiere/FCP/DaVinci）
PostPro 3.0: 指数剪辑     → AI 理解上下文，效率指数增长 ← 我们在这里
```

**关键公式**：Context 越丰富 → AI 加速效果越指数级增长

## 🔑 最核心的类比：编程 Agent vs 视频 Agent

创始人的核心论点：

```
编程领域:
  Code Generating Agent (基模生成代码) ← 不够
  Code-Editing Agent (Claude Code/Cursor) ← 真正需要的
  
视频领域:
  Video Generating Agent (Sora/可灵) ← 不够  
  Video-Editing Agent (Vattention) ← 真正需要的
```

> "基模负责生成，应用层 Agent 负责加工与编排"

**翻译成人话**：Sora 能生成视频片段，但把 100 个片段剪成一部成片 — 这才是 Vattention 要解决的问题。

## 🏗️ 三大引擎

| 引擎 | 名称 | 作用 |
|------|------|------|
| **MACE** | Meta Action Classification Engine | 训练数据处理：解析优质视频的创作模式 |
| **ACE** | Asset Comprehension Engine | 素材理解：端云协同的深度信息提取 |
| **PACE** | Predictive Action Chain Engine | 核心大脑：实时预测创作意图 + 智能操作引导 |

### Context Modeling vs Context Engineering

```
一键生成产品 (如 Sora):
  → Context Engineering（优化 Prompt 获取单次结果）
  → 够用

人机协作产品 (如 Vattention):
  → Context Modeling + Context Engineering
  → 持续理解、记忆、预测创作者意图
  → 类比 Claude Code 如何理解你的代码库
```

## 🦞 龙虾点评

### 1. 与 QCut 的关系

Vattention 的愿景和 QCut 高度重叠：
- 都是 **视频编辑 + AI Agent**
- 都强调 **Human-AI Cocreation**（人机协作，不是替代）
- 都关注 **素材→成片** 的加工环节

但切入角度不同：
- **QCut**: Electron 桌面端，AI 辅助时间线编辑
- **Vattention**: 从 Context Modeling 切入，强调预测和自主执行

### 2. "指数剪辑"概念的实质

本质上是说：AI 对你的项目理解越深 → 它能帮你做的越多 → 效率不是线性增长而是指数增长。这和 Claude Code "越了解代码库越好用"是同一个逻辑。

### 3. Video-Editing Agent vs Video-Generating Agent

这个区分非常精准。Sora/可灵解决的是"从无到有生成素材"，但专业创作者 90% 的工作是"把已有素材剪成成片"。后者才是真正的高价值场景。

### 4. 投资人阵容

青锐资本 + 百度风投 + 常垒资本，种子轮数百万美元。规模不大但组合有意思 — 百度风投意味着可能有大模型资源支持。
