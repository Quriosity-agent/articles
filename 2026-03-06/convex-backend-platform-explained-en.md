# What is Convex? Why It’s Called “The Backend Platform That Keeps Your App in Sync”

> **TL;DR**: Convex is not just a database and not just a generic BaaS. It combines a reactive database, TypeScript server functions, realtime sync, auth, scheduling, file storage, and search in one developer model. It is especially strong for realtime product patterns: chat, collaboration, and agent dashboards.

![Convex](convex-og.png)

## One-line Definition
Convex = reactive DB + TS functions + realtime synchronization.

## Core Capability Surface
- Functions (query/mutation/action/http)
- Database (schema, indexes, OCC, atomicity)
- Realtime data updates
- Auth integrations (OIDC/JWT/custom)
- Scheduling (cron and delayed jobs)
- File storage
- Search (text + vector)
- AI tooling hooks (MCP-related docs)

## Best Fit
- Frontend-led teams
- realtime apps (chat/collab/agent UI)
- fast iteration startups

## Less Ideal
- heavy SQL-centric architecture needs
- strict infra self-control requirements
- low tolerance for platform lock-in

## Why It Matters
Convex’s key value is reducing backend assembly overhead while preserving realtime UX quality.
