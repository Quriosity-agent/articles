# Claude 原生律所：一个两人团队如何用 AI 碾压百人大所

> **TL;DR**: 一位美国律师分享了他如何用 Claude 构建了一个两人精品律所，处理原本需要几十人团队的工作量。核心洞察：**专业法律 AI 产品（Harvey、Spellbook 等）不如配置好的通用 AI**。因为竞争优势不在模板库，而在律师个人的判断力 — 而 Claude 的 skill 系统恰好能编码这种判断力。最震撼的能力：Claude 能直接操作 Word 文档的 XML 层，生成真正的 tracked changes，不需要打开 Word。

---

## 🎯 核心论点：通用 AI > 垂直法律 AI

市场上有一堆法律专用 AI — Harvey、Spellbook、CoCounsel、Luminance。它们的共同论点是：律师需要专门为法律工作构建的 AI。

**作者的反驳：对于小型律所，配置好的通用 AI 更好。而且差距很大。**

原因：
- 这些产品都是同一个基础模型上的 wrapper
- **模板库不是竞争优势** — 每家律所的 NDA、股权购买协议都差不多
- 真正的差异化在于**律师个人的判断力**：怎么发现对方埋在第 14(c) 条的问题，怎么知道哪场赔偿之争值得打
- Claude 的 skill 系统能编码**个人判断力**，不只是公司模板

> "公司 playbook 和个人编码判断力的区别，就像给人一个菜谱和教人做菜的区别。"

## 🔧 三种模式

| 模式 | 用途 | 类比 |
|------|------|------|
| **Chat** | 分析法律问题、谈判策略、起草 | 跟聪明助理对话 |
| **Cowork** | 自主执行：审阅合同、生成文件包、批量编辑 | 放手让助理去干 |
| **Code** | 开发工具（作者因阅读困难，让 Claude 开发了法律文档转语音工具） | 程序员模式 |

**Cowork 是改变一切的模式** — 指向一个文件夹，给任务，Claude 自己读文件、创建新文件、编辑文档、做决策。

## 💎 杀手级能力：XML 级别的文档操作

**Claude 不是在"讨论"文档，而是能直接"操作"文档。**

Claude 可以：
- 在 .docx 的 XML 层面直接写 tracked changes markup
- 归属到律师本人名下
- 保留所有格式细节
- 对方律师在 Word 里打开看到的就是正常的修订标记

> "这就是给你一个能告诉你合同哪里有问题的助理，和一个能**修好问题、格式化、生成红线版、起草封面邮件**的助理的区别。"

专业法律 AI 给你一个能讨论文档的聊天机器人。Claude 是一个能**进入文档内部修改它们**的系统。

## 📋 实战案例

### 案例 1：收购交易深夜紧急谈判

- 晚上 7 点，买方律师发来最后通牒要求重构关键交易条款
- 上传购买协议 + 披露附表 + 需求函
- Claude 几分钟内发现：买方提出的两个 carve-out **直接矛盾**他们已确认的披露附表
- 第三个修改实际上会**削弱买方自己的**交割后保护
- 整晚谈判中持续投喂每封邮件，实时追踪每个让步的连锁影响
- 晚上 11 点完成全套反击方案
- **原本需要三个助理工作到凌晨的分析，两小时搞定**

### 案例 2：无幻觉的法律研究

- 客户需要了解新产品的监管格局（跨多个机构）
- Skill 指示 Claude **并行**研究所有相关角度（证券、州许可、银行、消保）
- **关键：交付前强制自审** — 验证每个引用的权威来源确实如所述，标记低置信度内容，检查内部矛盾，防范幻觉引用
- 那些因 AI 虚假引用被制裁的律师，问题不是 AI 本身，是没有质量控制

### 案例 3：40 页合同的完整红线审阅

- 对方发来 40 页修改版
- 上传 → 合同审阅 skill 自动触发
- 按严重度分类所有修改，标记风险转移，识别条款间冲突
- 决策后，Claude 直接生成 Word tracked changes
- **从收到标记到回复包准备就绪：不到 1 小时，其中 30 分钟是自己思考**

## 🧠 Skill 系统：编码判断力

作者让 Claude 分析了他们几个月的所有对话，识别出最高影响的 skill：

**6 个生产级 skill：**
1. **合同审阅** — 4 种模式、严重度评级、缺失条款检查、市场基准
2. **Tracked Changes 编辑** — XML 级别操作 Word 文档
3. **合同起草** — 从条款清单生成完整文件包
4. **客户沟通** — 正确语气的封面邮件和建议函
5. **法律研究** — 并行研究 + 自审验证 + 反幻觉
6. **政策撰写** — 监管合规文档

> **关键洞察：Skill 可以转移。** 如果有 50 个助理，安装到每台机器上，每个人立刻用你的分析框架、你的写作风格、你的格式偏好工作。**需要数年 mentor 才能传递的知识，现在是一个指令文件。**

## ⚖️ 特权和伦理

- ABA 指导意见和州律师协会伦理意见：AI 工具作为第三方技术提供商，受代理/工具例外覆盖
- Anthropic 提供零数据保留 API 和商业数据处理协议
- 作者起草了 AI 使用条款加入委托书，客户签字时眼都不眨
- **在大多数司法管辖区，技术能力现在是执业道德要求的一部分**

## 💰 商业影响

**人员配置：** 两人律所处理大所级别的工作量。传统需要雇助理的工作（初审、研究备忘、初稿、红线摘要、例行通信）现在由 Claude 在监督下完成。

**计费：** 订阅定价 + 传统小时计费混合。订阅客户获得持续顾问、合同审阅、合规监控，固定月费。客户不怕打电话，收入可预测。

**判断力的价值：**
> "有 10-20 年经验的律师坐在一个 AI 让其**更有价值而非更不值钱**的资产上，但大多数人没意识到。"

## 💭 为什么这很重要

这不只是一篇"律师用 AI"的故事。它揭示了几个深层趋势：

1. **通用 AI > 垂直 AI** — 前沿模型的每个新能力第一天就到你手里，wrapper 要等产品团队决定下一步做什么
2. **Skill/Plugin 是新的知识管理** — 不是文档库，是编码后的专业判断力
3. **XML 级文档操作是杀手锏** — 不是聊天，是实际修改文件
4. **小团队 + AI = 碾压大团队** — 成本结构差异让大所模式面临根本性挑战
5. **"不用 AI"比"用 AI"更难在职业责任上站住脚** — 伦理规则现在要求技术能力

**核心公式：前沿通用模型 + 个人判断力编码 + 自主执行模式 = 10x 杠杆**
