# Karpathy 的 Autoresearch 首战告捷：小模型上找到的 20 个改进，全部迁移到大模型

*2026-03-10*

上周末我们报道了 Karpathy 发布 [autoresearch](https://github.com/karpathy/autoresearch) 的消息——一个让 AI agent 自动进行机器学习研究的"配方"。当时很多人觉得这个想法很酷但还停留在概念阶段。

三天后，Karpathy 就交出了第一份成绩单。**结果比预期的还要好。**

## 实验：2 天，20 个改进，全部可叠加

Karpathy 让 autoresearch 在他的 [nanochat](https://github.com/karpathy/nanochat) 项目上跑了大约两天，目标是调优一个 depth=12 的小模型。Agent 自动找到了约 20 个能降低验证损失的改动。

关键发现：

- **所有 20 个改动都是可叠加的（additive）**——不是互相冲突的局部最优，而是可以全部堆叠在一起
- **全部迁移到了更大的 depth=24 模型**——在小模型上发现的改进，放到大模型上同样有效

第二点才是真正让人兴奋的地方。

## 为什么"向上迁移"是大事

正如社区用户 @CorvusLatimer 精辟总结的：**"小模型的自我改进能向上迁移，这是复利优势，不是演示把戏。"**

@northStar0x7 的评论更加直白：Karpathy 手动调了 nanochat 很久，以为已经调得差不多了。结果一个 agent 跑两天，就轻松找到 20 个真实的、可叠加的、可迁移的改进——包括 QK norm 上缺失的可学习缩放因子和 value 正则化这样的结构性发现。

这意味着你可以用便宜的小模型做实验，然后把发现直接应用到昂贵的大模型上。实验成本大幅降低，研究速度大幅提升。

## 社区争论：超参搜索还是真正的研究？

并非所有人都同样兴奋。

芬兰研究者 @a_karvonen 泼了点冷水：**"虽然我预期类似方法未来会有很好的结果，但目前这看起来只是一个新的超参数调优算法。"**

@EthanHe_42 则给出了一个有趣的定位：**"让我想起了 AutoML 和神经架构搜索。但这次有了智能。"**

这个争论触及了一个根本问题：当 AI agent 发现"QK norm 上缺少可学习的缩放因子"时，这算超参调优还是架构洞察？答案可能取决于你对"研究"的定义有多严格。但无论如何，**结果是实实在在的验证损失下降**。

## 相关工作

社区也提到了几个相关方向：

- **ResearchGym**（[arxiv:2602.15112](https://arxiv.org/abs/2602.15112)）——@aniketthh 提到的自动化研究框架
- **进化 Agent for NanoGPT**（[arxiv:2601.10657](https://arxiv.org/abs/2601.10657)）——@Minghao__Yan 提到的用进化策略优化 NanoGPT 的工作

@PopVerseYT 还注意到一个有趣的模式：改进的分布不是均匀的，而是呈现出"小步慢走然后突然跳跃"的模式——这暗示 agent 可能在探索和利用之间自然形成了某种节奏。

## Autoresearch 不是工具，是思路

值得强调的是 Karpathy 自己的说法：autoresearch **不是一个你直接"使用"的工具**，而是一个配方/思路——把它交给你的 agent，应用到你关心的问题上。

这个定位很聪明。它不是要做一个通用的 AutoML 平台，而是提供一个可复制的研究范式：**让 AI agent 在小模型上系统性地搜索改进，然后迁移到大模型。**

两天，20 个可迁移的改进。这个效率足以让任何做深度学习研究的人认真考虑把 agent 加入自己的工作流了。
