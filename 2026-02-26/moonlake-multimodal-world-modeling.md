## 什么是多模态世界模型？

当人们谈论"世界模型"时，通常指的是下一帧预测。但 Moonlake 认为这远远不够。

以一个保龄球瓶为例，它**同时**是：
- 空间中的一个带纹理的物体（视觉）
- 一个有质量和惯性的刚体（物理）
- 一个可以被击倒的对象（交互）
- 计分系统中的符号贡献者（逻辑）
- 碰撞时的声源（音频）

当保龄球击中球瓶的那一刻，这些模态必须**同步更新**。如果只有视觉变了但物理没反应，或者分数没更新，世界就变得不连贯了。

**核心理念：真正的世界模型必须维护跨模态的状态空间，并在动作发生时同步预测所有模态的状态转移。**

## 10 阶段自主构建流程

Moonlake 用保龄球游戏展示了 Agent 的完整推理轨迹：

### Phase 1：资产实例化
Agent 推理："保龄球游戏需要球道、球瓶、球和回球道。"自动生成所有 3D 资产，匹配赛博朋克美学。

### Phase 2：物理化
为球瓶和球添加 `RigidBody3D`，实现半真实的碰撞交互。

### Phase 3：空间布局 & UI
结构化摆放球道、边界，添加全息计分板。

### Phase 4：核心游戏逻辑
**关键步骤**——物理结果（球瓶倒下）必须更新符号状态（分数）。创建 `bowling_controller` 统一管理。

### Phase 5：球生命周期
每轮投球后球平滑返回、重新就位。

### Phase 6：边界稳定化
添加 `PinBarrier` 防止球瓶飞出球道，重置动画流畅自然。

### Phase 7：边界情况处理
没击中显示"GUTTER"，踩线显示"FOUL"，异常行为不再破坏游戏。

### Phase 8：音频集成
`AudioStreamPlayer3D` 实现空间音频——球滚动声、碰撞声、球瓶各自从物理位置发出声音。

### Phase 9：IK 集成
`TwoBoneIK3D` 实现自然的拾取动作——角色伸手、对齐、抬起，物理可信。

### Phase 10：打磨（用户反馈驱动）
Strike 特效强化、分数动画、重置过渡——从"能玩"变成"好玩"。

## 为什么值得关注？

1. **不是下一帧预测，是完整世界构建** — Agent 不只是生成画面，而是同时处理视觉、物理、逻辑、音频、交互五个模态
2. **符号抽象 + 多模态推理** — 通过符号抽象（球瓶是"可击倒对象"而非"一堆像素"）来统一不同模态
3. **10 步从零到可玩** — 展示了 AI 从资产生成到游戏打磨的完整创作链路
4. **Beta 已开放** — 可以在 app.moonlakeai.com 注册体验

这可能是**游戏开发的下一个范式**：不写代码，不画模型，描述你想要的世界，Agent 帮你构建。
